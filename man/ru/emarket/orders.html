На этой вкладке представлен список текущих заказов, зарегистрированных на сайте.

Список заказов также может быть экспортирован и импортирован в формате CSV.

Фильтрация заказов

Весь список может быть отфильтрован и отсортирован по любому из столбцов. Для добавления столбца нужно кликнуть на пиктограмму в правом углу. Набор добавляемых столбцов задается базовыми типами модуля.

Статусы заказов

В зависимости от того, на каком этапе находится в данный момент заказ, ему может быть присвоен различный статус.

Показать возможные статусы

  • Ожидает проверки — после того как покупатель оформил заказ, заказ появляется в списке со статусом «Ожидает проверки».
  • Отменен — покупателю может аннулировать со своей стороны сделанный им заказ. Для этого ему необходимо нажать на соответствующую кнопку напротив заказа на странице его истории заказов.
  • Отклонен — если товар закончился на складе, или есть ошибки в оплате, администратор выставляет заказу статус «Отклонен». При этом он остается в списке заказов и позднее ему можно поменять статус.
  • Принят — этот статус означает, что один из администраторов ресурса подтверждает реальное наличие данного товара, правильность оплаты и, как следствие, возможность отгрузки товара покупателю.
  • Готов — этот статус выставляется администратором и означает, что товар доставлен, оплачен и можно закрывать заказ как выполненный. Тем не менее заказ останется в списке.

Статусы оплаты

В зависимости от выбора способа оплаты и этапа, оплате может быть присвоен различный статус. Для оплаты, производимой при помощи платежной квитанции, актуален только статус «Принята», который в этом случае выставляется вручную.

Показать возможные статусы

  • Инициализирована — в случае электронных способов оплаты этот статус означает, что пользователь выбрал один из способов оплаты и нажал «Продолжить» для перехода к вводу данных оплаты.
  • Отклонена — если в процессе проверки электронной оплаты произошли какие-либо ошибки, будет выставлен этот статус.
  • Подтверждена — этот статус появляется только при оплате при помощи Яндекс.Деньги и означает, что проверка сервером Yandex данных об оплате прошла успешно. Этот статус будет автоматически заменен на «Принята».
  • Принята — этот статус выставляется автоматически платежной системой и означает, что оплата прошла успешно. В случае платежной квитанции администратор выставляет этот статус вручную.

Статусы доставки

Эти статусы выставляются администратором вручную в зависимости от этапа доставки заказа покупателю.

Начиная с версии 2.8, можно редактировать свойства элементов непосредственно из таблицы со списком: для этого наведите курсор на поле, значение которого необходимо изменить, и нажмите на всплывающую пиктограмму . Для сохранения изменений нажмите Enter, для отмены изменений — Esc.
